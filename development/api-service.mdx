nHungry communicates with a backend REST API hosted at `https://infoinnova.es/lolo/api`. All requests and responses use `application/json`. The API client is an axios-based singleton (`apiService`) defined in `services/apiService.ts`. It attaches a `Bearer` token to every request via a request interceptor, queues failed requests offline, and clears the session on a `401` response.

<Note>
  All passwords are **MD5-hashed on the client** before being sent to the server. The raw password never leaves the device. The `md5` npm package is used for hashing.
</Note>

## Base URL

```text
https://infoinnova.es/lolo/api
```

## Authentication

Endpoints marked **requires auth** expect a `Bearer` token in the `Authorization` header. The token is obtained from the `/login` or `/register` response and stored in the `loginData` state. The axios interceptor reads the token from `localStorageService` and injects it automatically â€” you do not need to set the header manually when using `apiService`.

```typescript
// Header injected automatically by the request interceptor
Authorization: Bearer <token>
```

## Endpoints

### POST /login

Authenticates an existing user and returns a session token.

<ParamField body="email" type="string" required>
  The user's email address.
</ParamField>

<ParamField body="pass" type="string" required>
  The user's password, MD5-hashed.
</ParamField>

<ParamField body="fingerID" type="string" required>
  The device fingerprint generated by `useFingerprint()`. Used by the server to associate the session with a device.
</ParamField>

<ResponseField name="result" type="boolean" required>
  `true` if authentication succeeded.
</ResponseField>

<ResponseField name="token" type="string">
  JWT Bearer token. Present only when `result` is `true`.
</ResponseField>

<ResponseField name="error_msg" type="string">
  Human-readable error message. Present only when `result` is `false`.
</ResponseField>

**Example request**

```typescript
import md5 from 'md5';
import { apiService } from '~/services/apiService';

const response = await apiService.login(
  'user@example.com',
  md5('MyPassword1').toString(),
  store.loginData.value.fingerID
);

if (response.result) {
  store.setToken(response.token);
  store.setLogged(true);
}
```

**Example response (success)**

```json
{
  "result": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Example response (failure)**

```json
{
  "result": false,
  "error_msg": "Invalid credentials"
}
```

### POST /register

Creates a new user account and returns a session token.

<ParamField body="email" type="string" required>
  The email address for the new account.
</ParamField>

<ParamField body="pass" type="string" required>
  The desired password, MD5-hashed. The raw password is validated client-side (min 8 characters, one uppercase, one lowercase, one digit) before hashing.
</ParamField>

<ParamField body="fingerID" type="string" required>
  The device fingerprint.
</ParamField>

<ResponseField name="result" type="boolean" required>
  `true` if registration succeeded.
</ResponseField>

<ResponseField name="token" type="string">
  JWT Bearer token for the new session. Present only when `result` is `true`.
</ResponseField>

<ResponseField name="error_msg" type="string">
  Human-readable error message. Present only when `result` is `false`.
</ResponseField>

### GET /getAll

Retrieves all data for the authenticated user â€” products, categories, and supermarkets.

**Requires auth.** No request body. The Bearer token in the `Authorization` header identifies the user.

<ResponseField name="result" type="boolean" required>
  `true` if the request succeeded.
</ResponseField>

<ResponseField name="data" type="object">
  The user's complete dataset. Present when `result` is `true`. Contains `productos`, `categorias`, and `supermercados` arrays.
</ResponseField>

<ResponseField name="error_msg" type="string">
  Error message. Present when `result` is `false`.
</ResponseField>

**Example**

```typescript
import { apiService } from '~/services/apiService';

const response = await apiService.getAll();
if (response.result) {
  await store.importData(response.data);
}
```

### POST /syncData

The primary sync endpoint. Sends local data to the server, which merges it with any data stored from other devices and returns the reconciled result. On success the client imports the returned data, replacing the local state.

**Requires auth.**

<ParamField body="email" type="string" required>
  The authenticated user's email.
</ParamField>

<ParamField body="token" type="string" required>
  The current Bearer token (sent in the body in addition to the header).
</ParamField>

<ParamField body="fingerID" type="string" required>
  The device fingerprint.
</ParamField>

<ParamField body="data" type="object" required>
  The local dataset to sync. Produced by `store.exportData()`. Contains `productos`, `categorias`, `supermercados`, and metadata fields. `loginData` is explicitly removed before sending.
</ParamField>

<ResponseField name="result" type="boolean" required>
  `true` if the sync succeeded.
</ResponseField>

<ResponseField name="data" type="object">
  The merged dataset returned by the server. Import it with `store.importData()`.
</ResponseField>

<ResponseField name="token" type="string">
  A refreshed Bearer token, if the server issues one.
</ResponseField>

<ResponseField name="error_msg" type="string">
  Error message. Present when `result` is `false`.
</ResponseField>

**Example**

```typescript
import { apiService } from '~/services/apiService';
import { myStore } from '~/composables/useStore';

const store = myStore();
const dataToSync = store.exportData();
delete (dataToSync as any).loginData; // never send credentials

const response = await apiService.syncData(
  store.loginData.value.email,
  store.loginData.value.token,
  store.loginData.value.fingerID,
  dataToSync
);

if (response.result && response.data) {
  store.importData(response.data);
  store.updateLastSyncTimestamp();
}
```

## Error handling

### HTTP 401 Unauthorized

The request interceptor detects a `401` response and clears the local session:

```typescript
// Handled automatically by apiService interceptors
localStorageService.setSubItem('loginData', {
  email: '',
  token: '',
  fingerID: '',
  logged: false
});
```

The user must log in again to obtain a new token. See [register and log in](/account/registration-login).

### Network errors and offline queue

When a request fails because there is no network connection (no `response` object on the error), `apiService` adds the request to an in-memory queue and persists it to localStorage under the key `pendingRequests`:

```json
[
  { "method": "post", "endpoint": "/updateProducto", "data": { "id_producto": 7, "done": 1 } }
]
```

When the browser fires the `online` event, `apiService` processes the queue automatically, replaying each request in order. If a queued request fails again due to a non-network error (e.g., `4xx`), it is not re-queued.

<Warning>
  The offline queue stores only the method, endpoint, and data â€” not the resolve/reject callbacks. If the page is reloaded before the queue is drained, the promises that were waiting are lost. The queue is replayed on the next page load when connectivity is detected.
</Warning>

### Timeout

All requests have a 10-second timeout. Requests that exceed this limit are treated as network errors and follow the offline queue path described above.
